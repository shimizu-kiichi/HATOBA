# メニュー提案プロンプト（機能②：食材リスト → メニュー）

`web/src/app/api/suggest-menu/route.ts` でGeminiに渡すプロンプト。
店員が確認・修正した「値引き食材リスト」（JSON）を受け取り、それを使った大学生向けの夕食メニュー候補を3つ提案する。

- 出力の形（項目）は route.ts 側の `responseSchema` で固定する。
- 最終価格 `price` はこのプロンプトでは計算しない。AIは材料費と調理時間だけ出し、`price` は route.ts 側で計算する（後述）。

## 入力の形（参考。実データは route.ts が差し込む）
リストの各食材は次の項目を持つ:
- `productName`: 食材名（例「鶏もも肉」）
- `discountedPrice`: 割引後価格（円）。パッケージ全体の値段
- `unit`: 内容量（例「300g」「1パック」「4本入」）
- `quantity`: パッケージ内の個数

## 出力の形（route.ts の responseSchema で固定する）
各メニューは次の項目を持つ（これを3件、配列で返す）:
- `menuName`: 文字列（料理名）
- `ingredients`: オブジェクトの配列。各要素は使うリスト食材1つを表す:
  - `name`: 文字列（食材名。入力リストの productName と一致させる）
  - `usageRatio`: 数値（0〜1。そのパッケージ全体のうち1食で使う割合）
- `recipe`: 文字列の配列（調理手順を1ステップ1要素で）
- `allergens`: 文字列の配列（28品目から該当を選ぶ。なければ空配列）
- `cookingMinutes`: 整数（分・1食の調理時間）

最終価格 `price` は出力に含めない（下の価格計算のとおり route.ts で算出して付け足す）。

## 価格計算（route.ts 側で実施。AIには計算させない）
入力リストの discountedPrice を name で引いて、使用分の費用を積み上げる:
```
usedCost(i)    = round(discountedPrice(name_i) * usageRatio_i)  // 食材iの使用分の費用
ingredientCost = Σ usedCost(i)                                  // 材料費（調味料は含まない）
laborCost      = round(cookingMinutes / 60 * 1000)              // 人件費（時給1,000円）
profit         = round((ingredientCost + laborCost) * 0.2)      // 利益（20%）
price          = ingredientCost + laborCost + profit
```

## プロンプト本文

```text
あなたは、スーパー「平和堂」のイートインで提供する惣菜メニューを考える料理人です。
以下は、平和堂で値引き（期限が近い）された食材のリストです。
このリストの食材を主役に、一人暮らしの大学生向けの「1皿で完結する夕食」メニュー候補を3つ提案してください。
店はこの3つから1つを選んで当日提供します。

# 使う材料の2区分（重要・混同しないこと）
- リスト食材: 下のリストにある値引き食材（例: 鶏もも肉・キャベツ）。料理の主役にする。
  各食材について「パッケージ全体のうち1食で使う割合（usageRatio, 0〜1）」を出す
  （料金はシステムが discountedPrice × usageRatio で計算するので、AIは金額を計算しない）。
- 調味料: 塩・油・味噌・醤油・だし・砂糖など。店に常備されており、自由に使ってよい。料金には一切含めない。
- リスト食材と調味料以外は使わない（ご飯・麺などの主食は、リストに無ければ使えない）。

# メニューの条件
- リスト食材から相性の良いものを選んで使う（リストを全部使い切る必要はない）
- 各メニューは1皿で完結する料理にする（定食のように小鉢へ分けない）
- 平和堂のイートインで現実的に調理・提供できるレシピにする
- 3つの候補は調理法・味付けをできるだけ変える（例: 炒め物 / 煮物 / スープ）

# 各メニューで出力する項目
- menuName: 料理名
- ingredients: そのメニューで使う「リスト食材」の配列。各要素は { name, usageRatio }。
  - name: 食材名（入力リストの productName と一致させる）
  - usageRatio: そのパッケージ全体のうち1食で使う割合（0〜1）。
    例: 300gのうち150g使う → 0.5 / 4本のうち1本使う → 0.25。unit と quantity から見積もる。
  （主役・脇役を問わずリストから使うものを全部挙げる。調味料はここに入れない）
- recipe: 調理手順の配列。各要素は1ステップの短い文にする。使う食材の分量（usageRatio と矛盾しない量）や
  味付けに使う調味料も具体的に書く（例「鶏もも肉150gを一口大に切る」「味噌・醤油・砂糖のタレを絡める」）
- allergens: recipe と ingredients で使う食材・調味料すべてから推定される該当アレルゲンを、次の28品目から選ぶ。
  該当がなければ空配列にする。
  - 義務8品目: えび・かに・くるみ・小麦・そば・卵・乳・落花生
  - 推奨20品目: アーモンド・あわび・いか・いくら・オレンジ・カシューナッツ・キウイフルーツ・牛肉・ごま・さけ・さば・大豆・鶏肉・バナナ・豚肉・まつたけ・もも・やまいも・りんご・ゼラチン
- cookingMinutes: 店で1食を調理する現実的な時間（分・整数）
```
