# カスタマー画面の料理画像（AI生成イメージ → 実写差し替え）

> 客側（costomer）画面に出す料理写真の扱いをまとめる。
> 店が料理を完成・撮影するまでの「つなぎ」としてAI生成のイメージ画像を表示し、完成後に実写へ差し替える。

## 目的・背景

公開直後はまだ実際の料理写真が無い。空欄やプレースホルダーのままだと客側に魅力が伝わらないので、
**料理名・レシピから生成したAIイメージ画像を仮に表示**しておき、店が実際に作って撮影したら**実写に差し替える**。

## 全体の流れ

```
[店側] AIがメニューを提案 → 料理名・レシピが確定（一時保存）
        │ 料理名・レシピ
        ▼
[画像生成AI] 料理名＋レシピをプロンプトとして投げる
        │   ・処理場所: 客側フロント（costomer の中）
        │   ・生成手段: 無料の画像生成API（Pollinations.ai / キー不要）
        ▼
[客側画面] 生成画像を料理写真エリアに表示
        │   ・「イメージ画像」であることを明記（AI生成・実物と異なる旨）
        ▼
（ここまでは仮の状態。店が料理を完成させるまでのつなぎ）
        │
        ▼
[店側] 料理が完成 → その場で写真を撮影 → アップロード
        ▼
[客側画面] 以降は実写を表示（「イメージ画像」表記を外す）
```

## ステップ詳細

1. **料理名・レシピを取得**
   店側AIの出力（料理名・レシピ）を一時保存し、それを画像生成のプロンプト材料にする。

2. **画像をAI生成する**
   - 処理は**客側フロント（costomer 内）**で行う。
   - 生成手段は**無料の Pollinations.ai**（APIキー不要）を想定。
     `https://image.pollinations.ai/prompt/{プロンプト}?width=...&height=...&model=flux`
   - プロンプトは「料理名 ＋ レシピ要素 ＋ 写真風の指定（food photography, appetizing 等）」で組み立てる。

3. **客画面に表示し、イメージ画像であることを明記**
   - 生成画像を料理写真エリアにはめ込む。
   - 画像上またはキャプションに「**※イメージ画像（AI生成）**」と明記し、実物と異なる可能性を伝える。

4. **これは「仮」の状態**
   - あくまで店が料理を完成させるまでのつなぎ。

5. **実写への差し替え**
   - 店側に「**完成した料理の写真を撮影・アップロード**」する機能を用意する。
   - アップロードされたら、客画面の表示をその実写に切り替え、「イメージ画像」表記を外す。

## 状態の整理

| 状態 | 表示する画像 | 表記 |
|--|--|--|
| 仮（公開直後〜調理中） | AI生成イメージ | 「※イメージ画像（AI生成）」 |
| 確定（料理完成・撮影後） | 店が撮った実写 | 表記なし |

### 表示ロジック（重要）

「※イメージ画像（AI生成）」の表記は、**表示している画像がAI生成かどうかに連動**させる。
- 画像が **AI生成イメージ** のとき → 表記を**出す**
- 画像が **店の実写に差し替わった** とき → 表記を**自動で消す**

つまり表記は画像の種類にひも付け、**実写を挿入した瞬間に表記も消える**実装にする
（例: メニューデータに `imageIsReal`（または実写URLの有無）を持たせ、それで表記の出し分けを判定する）。

## 技術メモ・注意点

- **画像生成API**: Pollinations.ai は無料・キー不要・写真風。Gemini とは別系統（ただの fetch / URL）なので、Gemini APIキーの状態とは無関係に動かせる。
- **「イメージ画像」明記は必須**: AI生成は実物と異なるため、誤認防止の表記を必ず入れる。
- **生成のブレ・遅延対策**: 無料公開サービスはレート制限や遅延があり得る。デモでは**事前に1回生成して保存（キャッシュ）**しておくと安定する。
- **一時保存の置き場所**: 店側→客側の「公開メニュー保存」と一体化させると自然（メニュー情報と一緒に画像／画像URLも持たせる）。

## 今後の検討事項

- 一時保存の具体的な置き場所（メモリ / ファイル / KV）。
- 実写アップロードの保存先（`web/public/` か外部ストレージか）。
- 実写に切り替わったとき、客画面へどう反映するか（再読み込み / 自動更新）。
