# HATOBA デモ動画 絵コンテ（v2）

> 最終成果物は **発表用デモ動画**（リアルタイムデモ禁止）。
> この絵コンテから逆算して画面・機能を作る。二人で叩き台として修正していく。

## 狙い（30秒で伝える価値）
夕食難民の大学生 × 平和堂イートイン。
**「期限の近い食材から、AIが今日の格安メニューを作り、学生が席で注文して食べる」** という一連の流れを見せる。

## 撮影方針（v2 決定）
- **実写**: スマホで食材を撮影し画像認識するシーン（＝つかみの主役）。リアルな手元・店内で「本当に動いてる感」を出す。
- **画面録画**: それ以降のAIメニュー提案〜注文〜決済はアプリ画面の録画でOK。
- **6/20 中間報告会のゴール**: **シーン2（食材写真→AIメニュー提案）が実際に動く**こと。ここが最優先。

## 具体シナリオ例（同じ画を想像するための叩き台 ※平和堂の実商品に合わせて差し替え）
- **シーン1で撮る余り食材**: 鶏もも肉 / キャベツ / 人参（閉店前で値引きシール付き）
- **シーン2でAIが提案するメニュー例**:
  - 「鶏とキャベツの味噌だれ丼」 ¥420（アレルギー: 小麦・大豆）
  - 「鶏と野菜のあんかけ焼きそば」 ¥450（アレルギー: 小麦・卵）
  - 「チキンと春キャベツのスープごはん」 ¥380（アレルギー: 乳）
- **シーン3で確定するメニュー**: 「鶏とキャベツの味噌だれ丼」
- **シーン4で学生が見る表示**: 本日のメニュー「鶏とキャベツの味噌だれ丼 ¥420」／残り 8 食

## 「本物 / モック」の切り分け
- **本物にする = シーン2（食材写真→AIメニュー提案）のみ。** ここだけ実際にAIを動かす。
- それ以外の画面はモック（固定データ・ボタンで遷移するだけ）でOK。動画で動いて見えれば満点。
